# 01 · Pre-Demo Checklist — Kokoro Sushi Roma

> **INTERNO / BNS STUDIO** — non condividere con il cliente.
> Demo: **https://demo.kokoro.bnsstudio.it**

---

## ✅ Prima di uscire

- [ ] MacBook carico (≥ 90%)
- [ ] Caricatore MacBook nello zaino
- [ ] Hotspot / SIM dati attivi e testati
- [ ] Smartphone carico (per la demo mobile davanti al cliente)
- [ ] Power bank
- [ ] Demo online raggiungibile: apri https://demo.kokoro.bnsstudio.it
- [ ] Test mobile eseguito stamattina (menu, login, prenotazione)
- [ ] Credenziali demo salvate offline (vedi `06_TECHNICAL_DEMO_HANDOVER.md`)
- [ ] QR code della demo stampato + salvato sul telefono
- [ ] Contatti del referente / numero del locale
- [ ] Proposta economica pronta (a voce o su foglio, non nella demo)
- [ ] Biglietti da visita BNS Studio

---

## ✅ Prima di entrare (in auto / davanti al locale)

- [ ] Aprire la demo su MacBook **e** su smartphone
- [ ] Verificare connessione (hotspot se il Wi-Fi del locale è lento)
- [ ] Effettuare login demo (cliente **e** ristoratore) per "scaldare" le pagine
- [ ] Testare una prenotazione veloce e poi ricaricare
- [ ] Chiudere tutte le tab inutili del browser
- [ ] Attivare **Non disturbare** su Mac e telefono (niente notifiche personali)
- [ ] Luminosità schermo al massimo
- [ ] Zoom browser al 100%
- [ ] Preparare lo smartphone con la home della demo già aperta

---

## ✅ Durante la demo — sequenza pagine

Ordine consigliato (dettaglio in `02_DEMO_SCRIPT.md`):

1. [ ] **Home** — hero, brand, atmosfera (da smartphone)
2. [ ] **Menu mobile** — apri il menu hamburger: mostra l'esperienza editoriale
3. [ ] **Menu digitale** (`/menu`) — categorie, ricerca, filtri, allergeni
4. [ ] **Prenotazione** (`/prenota`) — data → orario → dati → conferma
5. [ ] **Area cliente** — login `cliente@kokoro.it`, prenotazioni + notifiche
6. [ ] **Pannello gestione** — login `admin@kokoro.it`, calendario e CMS menu
7. [ ] **Ritorno alla Home** — chiudere con il valore complessivo
